# Unblinded

Este proyecto, desarrollado por Axel Valton Juan y Joan Merlos Cremades, ha consistido en el desarrollo de un software que permita leer texto, describir paisajes, guiar hacia una determinada dirección y buscar objetos con la intención de hacer más sencillo el día a día de las personas invidentes. Para que este proyecto fuera concluido de forma adecuada se deberian implementar estas funciones en un dispositivo que tuviese cámara, altavoces, micrófono y gps. No obstante, en nuestro caso nos hemos limitado a desarrollar el código para que sea funcional en un ordenador convencional.

Los archivos de código desarrollado son:

* `Unblinded_Script.py`: es el Script principal del código que nos permite elegir entre los diferentes modos de nuestro dispositivo, siendo estos **Lectura**, **Descripción**, **Navegación guiada** (se ha implementado un modo que simula el trayecto de una persona a través de una ruta dada) y **Búsqueda** (que debe ser usado en conjunto con el archivo `Unblinded_busqueda.ipynb` que nos permite obtener la foto tratada).

* `Unblinded_Functions.py`: es el Script secundario que contiene todas las funciones que permiten que el programa funcione. Cabe destacar que estas funciones utilizan diferentes modelos como gTTS para pasar de texto a audio, Whisper para poder reproducir el audio, google Maps para obtener las rutas y diferentes modelos de la IA generativa `llama` para poder procesar tanto imágenes como texto.

* `Unblinded_busqueda.ipynb`: este archivo nos permite a partir de una imagen y un texto obtener una imagen que resalta los objetos mencionados en el texto, es decir, nos permite utilizar el modo búsqueda. Para utilizarlo primero debemos activar el modo búsqueda de `Unblinded_Script.py` siguiendo los pasos indicados hasta obtener por terminal una palabra en inglés. Luego debemos introducir dicha palabra en la variable *prompt* de este notebook en conjunto con la imagen que deseemos analizar. Una vez obtenida la imagen tratada la introducimos con el nombre correspondiente en el archivo de `Unblinded_Script.py` y continuamos con su ejecución con normalidad para obtener la solución deseada. Cabe destacar que este forma tan rudimentaria de realizar este modo se debe a que había una incompatibilidad entre el modelo SAM3, que es el que utilizamos para destacar los objetos del prompt en las imágenes y las librerías utilizadas en el script normal.
